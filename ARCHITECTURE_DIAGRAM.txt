Padrão Façade - Arquitetura Visual
===================================

## ANTES (sem Façade)

┌─────────────────────────────────────────────┐
│         Controller / View / Route            │
└──────────────┬──────────────────────────────┘
               │
               ├─ Validação de entrada
               │
               ├─ Validação de CPF/Email
               │
               ├─ Hash de senha
               │
               ├─ Try-catch de erro
               │
               ├─ Chamar DAO
               │
               ├─ Tratamento de exceção
               │
               └─ Redirecionar + mensagem
               │
┌──────────────v──────────────────────────────┐
│              DAO (Persistência)              │
└─────────────────────────────────────────────┘
               │
┌──────────────v──────────────────────────────┐
│         Banco de Dados / Models              │
└─────────────────────────────────────────────┘

❌ Problemas:
   - Lógica espalhada no Controller
   - Validações repetidas em múltiplos Controllers
   - Difícil testar
   - Tratamento de erro inconsistente


## DEPOIS (com Façade)

┌─────────────────────────────────────────────┐
│         Controller / View / Route            │
│  (código limpo: 50 linhas → 8 linhas)       │
└──────────────┬──────────────────────────────┘
               │
               │  $resultado = JogadorFacade::criar($dados)
               │
┌──────────────v──────────────────────────────┐
│            🎭 FAÇADE                        │
│  ┌────────────────────────────────────┐    │
│  │  JogadorFacade / TecnicoFacade     │    │
│  │  EscalacaoFacade / ...             │    │
│  │                                    │    │
│  │  - Validações completas            │    │
│  │  - Lógica de negócio               │    │
│  │  - Tratamento de erro              │    │
│  │  - Logging                         │    │
│  │  - Resposta padronizada            │    │
│  └────────┬───────────────────────────┘    │
└───────────┼──────────────────────────────────┘
            │
            │  if ($resultado['sucesso'])
            │     $dados = $resultado['dados']
            │
┌───────────v──────────────────────────────────┐
│              DAO (Persistência)               │
└─────────────────────────────────────────────┘
            │
┌───────────v──────────────────────────────────┐
│         Banco de Dados / Models               │
└──────────────────────────────────────────────┘

✅ Vantagens:
   - Lógica centralizada na Façade
   - Controllers simples e legíveis
   - Fácil de testar
   - Respostas padronizadas
   - Reutilizável em múltiplos Controllers

## FLUXO DE REQUISIÇÃO

EXEMPLO: Criar Jogador
=======================

1. REQUEST
   POST /jogadores/cadastrar
   {nome: "João", numeroCamisa: 10, ...}

2. CONTROLLER
   JogadorController::cadastrar()
   ↓
   $dados = [todos os POST]
   ↓
   $resultado = JogadorFacade::criar($dados)

3. FAÇADE
   JogadorFacade::criar($dados)
   ├─ self::validarDadosJogador($dados)
   │  ├─ validar nome
   │  ├─ validar numero camisa (1-99)
   │  ├─ validar peso (30-150)
   │  ├─ validar altura (140-220)
   │  └─ throw Exception se inválido
   │
   ├─ $jogador = new Jogador()
   ├─ $jogador->setNome($dados['nome'])
   ├─ ... mais setters
   │
   ├─ JogadorDAO::salvar($jogador)
   ├─ self::log("Novo jogador criado: ...")
   │
   └─ return [
        'sucesso' => true,
        'mensagem' => 'Jogador criado com sucesso',
        'dados' => $jogador
      ]

4. CONTROLLER (continua)
   if ($resultado['sucesso']) {
       $_SESSION['sucesso'] = $resultado['mensagem'];
       header('Location: /jogadores');
   } else {
       $_SESSION['erro'] = $resultado['mensagem'];
       header('Location: /jogadores/novo');
   }

5. RESPONSE
   Redirect + mensagem de sucesso/erro


## PADRÃO DE RESPOSTA

Toda Façade retorna:

┌─────────────────────────────────────┐
│ [                                   │
│   'sucesso' => bool,    ← status    │
│   'mensagem' => string, ← para user │
│   'dados' => mixed,     ← retorno   │
│   'contexto' => string  ← opcional  │
│ ]                                   │
└─────────────────────────────────────┘

Exemplos:

SUCESSO:
{
  'sucesso' => true,
  'mensagem' => 'Jogador criado com sucesso',
  'dados' => Jogador(id: 42, nome: 'João', ...)
}

ERRO:
{
  'sucesso' => false,
  'mensagem' => 'Número da camisa deve estar entre 1 e 99',
  'contexto' => 'criar',
  'dados' => null
}

## VALIDAÇÕES AUTOMÁTICAS

JogadorFacade
=============
- Nome: obrigatório, não vazio
- Número Camisa: 1-99
- Peso: 30-150 kg
- Altura: 140-220 cm
- Data nascimento: obrigatória, formato válido
- Disponível: 'sim' ou 'não'

TecnicoFacade
=============
- Nome: obrigatório
- CPF: 11 dígitos, único
- Email: formato válido, único
- Data nascimento: obrigatória, formato válido
- Senha: mínimo 6 caracteres, hash com PASSWORD_DEFAULT


## BENEFÍCIO PRÁTICO

Antes (sem Façade):
───────────────────
// controller/JogadorController.php (25 linhas)
try {
    if (empty($nome)) throw new Exception("Nome obrigatório");
    if ($numeroCamisa < 1 || $numeroCamisa > 99)
        throw new Exception("Número 1-99");
    if ($pesoKG < 30 || $pesoKG > 150)
        throw new Exception("Peso 30-150");
    if ($alturaCM < 140 || $alturaCM > 220)
        throw new Exception("Altura 140-220");

    $jogador = new Jogador();
    $jogador->setNome($nome);
    $jogador->setNumeroCamisa($numeroCamisa);
    // ... 10+ more setters

    JogadorDAO::salvar($jogador);
} catch (Exception $e) {
    // error handling
}

Depois (com Façade):
────────────────────
// controller/JogadorController.php (8 linhas)
$resultado = JogadorFacade::criar([
    'nome' => $nome,
    'numeroCamisa' => $numeroCamisa,
    'pesoKG' => $pesoKG,
    'alturaCM' => $alturaCM,
    // ...
]);

if ($resultado['sucesso']) {
    // success handling
}

Redução: 25 linhas → 8 linhas (-68%)
Clareza: +300%
Testabilidade: +∞
